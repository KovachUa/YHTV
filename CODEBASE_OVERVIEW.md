# YHTV Codebase Overview

Цей файл створено для швидкого розуміння архітектури проєкту YHTV. Він описує, де знаходиться ключовий функціонал, як працюють компоненти та як вони взаємодіють між собою.

## 🏗 Архітектура проєкту

Проєкт складається з клієнтської частини (Frontend) та серверної частини (Backend), які працюють у Docker-контейнерах разом із базами даних PostgreSQL (для збереження інформації) та Redis (для черги завдань Celery).

---

## 📂 Backend (Серверна частина)
Знаходиться у папці `backend/`. Написаний на Python з використанням FastAPI (API) та Celery (асинхронні завдання).

- **`backend/main.py`** 
  - Основний файл додатку FastAPI.
  - Містить всі API-ендпоінти: 
    - `/api/config` (управління налаштуваннями `yt-dlp`), 
    - `/api/download` (додавання відео у чергу), 
    - `/api/videos` (отримання списку та метаданих відео), 
    - `/api/channels` (робота з каналами), 
    - `/api/queue` (управління чергою завантажень).
  - Відповідає за читання/запис конфігурації у `config.json` та взаємодію з БД.

- **`backend/tasks.py`**
  - Містить логіку асинхронних завантажень за допомогою **Celery** та `yt-dlp`.
  - Функція `download_video_task`: налаштовує `yt-dlp` на основі `config.json`, запускає завантаження, збирає відсоток прогресу і записує статус у базу даних.
  - Тут вручну зберігаються метадані (`.info.json`) для опису та коментарів до відео.

- **`backend/models.py`**
  - Описує структуру таблиць бази даних за допомогою SQLAlchemy.
  - Основні моделі: `Channel` (Канали), `Video` (Завантажені відео), `DownloadTask` (Завдання у черзі/Історія).

- **`backend/database.py`** & **`backend/celery_app.py`**
  - Налаштування підключень до PostgreSQL бази даних та ініціалізація брокера Redis для Celery.

---

## 🎨 Frontend (Клієнтська частина)
Знаходиться у папці `frontend/`. Побудований на звичайних HTML, CSS та JavaScript (без важких фреймворків).

- **`frontend/index.html`**
  - Головна та єдина сторінка (SPA). Містить всі модальні вікна (додавання відео, налаштування, перегляд метаданих).

- **`frontend/static/js/app.js`**
  - Головний скрипт, який керує інтерфейсом.
  - Робить API-запити до бекенду (`fetch`), оновлює таблиці з чергою, відео та каналами.
  - Обробляє заповнення форм налаштувань та відображення модалок із коментарями.

- **`frontend/static/js/bg.js`** та **`frontend/static/js/lang.js`**
  - Відповідають за візуальні ефекти (фон) та систему локалізації (переклади інтерфейсу).

---

## 🐳 Інфраструктура (Docker)

- **`docker-compose.yml`**
  - Описує 5 сервісів:
    1. **`frontend`** (Nginx для роздачі статики, порт 8080).
    2. **`backend`** (FastAPI сервер, uvicorn).
    3. **`worker`** (Celery процес, який безпосередньо виконує завантаження).
    4. **`db`** (PostgreSQL 16 для збереження моделей).
    5. **`redis`** (Брокер повідомлень для Celery).
  - Створює volume `yhtv_downloads`, який монтується у бекенд та воркер для збереження медіа.

## 📥 Дані та завантаження
- **Папка `yhtv_downloads/` (або volume `yhtv_downloads`)**
  - Тут фізично зберігаються всі завантажені відео (`.mp4`, `.webm`, `.mkv`), обкладинки (`.webp`), файли конфігурації (`config.json`) та метадані (`.info.json`).

---

**Шпаргалка для АІ:** Якщо потрібно змінити логіку завантаження — дивись `tasks.py`. Якщо треба змінити API чи ендпоінти — `main.py`. Для UI — `index.html` та `app.js`.
